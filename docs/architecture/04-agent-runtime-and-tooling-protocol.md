# Agent 运行时与工具协议（V1 提纲）

## 1. 文档目的
定义去中心化社交系统中的 Agent 执行模型、工具调用协议和治理要求，确保三端行为一致。

## 2. Agent 运行时目标
- 让 Agent 在 Desktop、Mobile、Server 三端可协同执行。
- 让工具调用可控、可审计、可回放。
- 让执行策略可插拔并可按场景动态调整。

## 3. Agent 执行模型
## 3.1 生命周期
- Create：创建实例并绑定作用域（用户/群组/组织）。
- Plan：生成执行计划与工具序列。
- Execute：按策略执行任务与调用工具。
- Suspend/Resume：支持中断恢复。
- Terminate：安全结束并落盘审计。

## 3.2 执行位置策略
- 端侧优先：隐私敏感、低延迟、轻量任务。
- 云侧优先：高算力、跨节点协调、长时任务。
- 混合执行：计划在端，重算在云，结果回端。

## 4. 工具协议
## 4.1 工具模型
- Tool Definition：名称、输入 schema、输出 schema、权限声明。
- Tool Capability：网络、存储、系统能力、外部服务访问等级。
- Tool Policy：速率限制、超时、重试、幂等要求。

## 4.2 调用协议
- Request：trace_id、agent_id、tool_name、payload、timeout。
- Response：status、result、error_code、error_message、metrics。
- Event：执行进度、人工介入请求、异常告警。

## 4.3 失败语义
- 可重试错误：网络抖动、临时限流、依赖超时。
- 不可重试错误：权限不足、参数非法、策略拒绝。
- 兜底行为：降级、跳过、人工确认。

## 5. 记忆与上下文
- Session Memory：短期上下文，任务结束可过期。
- Long-term Memory：长期偏好与知识，受权限与隐私策略约束。
- Retrieval Policy：按作用域和敏感级别检索，禁止越权拼接上下文。

## 6. 安全与治理
- Agent 和 Tool 都要最小权限原则。
- 每次调用保留审计证据（输入摘要、输出摘要、决策路径）。
- 高风险工具强制二次确认或审批。
- 提供全局 Kill Switch 与单 Agent 熔断机制。

## 7. 可观测性
- 核心指标：成功率、平均时延、重试率、人工接管率。
- 调试能力：端云链路追踪、工具调用回放、失败根因聚类。
- 质量门槛：关键任务链路设置 SLO 与告警阈值。

## 8. 与 Applet 体系关系
- Applet 可以声明可被 Agent 调用的工具接口。
- Agent 调用 Applet 工具时仍走统一权限与审计链路。
- Applet 不得绕过 Agent 工具协议直接调用高风险宿主能力。

## 9. 待决策问题
- 是否允许第三方 Agent 运行时插件进入生产环境。
- 工具协议是否支持流式输出作为标准能力。
- 端侧模型与云侧模型的策略切换阈值如何定义。
