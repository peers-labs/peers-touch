# 数据、身份与同步模型

## 1. 文档目的
定义系统中身份、数据模型和跨端同步机制的统一语义，作为协议层与实现层的对齐基准。

## 2. 身份模型
## 2.1 实体
- User Identity：用户主身份，跨设备稳定。
- Device Identity：设备身份，和用户形成授权绑定关系。
- Agent Identity：Agent 执行体身份，可绑定用户或组织作用域。

## 2.2 关系
- 用户可绑定多个设备，设备可轮换密钥。
- Agent 以最小权限代表用户或组织执行任务。
- 所有身份关系变更形成可审计事件流。

## 3. 数据分层
- Profile Data：用户公开资料与偏好。
- Social Graph：关注、好友、群成员等关系数据。
- Conversation Data：会话、消息、状态回执。
- Agent Data：记忆、计划、工具调用结果、执行日志。
- Applet Data：扩展模块私有状态与配置。

## 4. 一致性策略
### 4.1 一致性目标
- 用户可感知路径保持“读写可用优先”。
- 跨端状态在可接受时间窗内收敛。
- 冲突可解释、可重放、可追责。

### 4.2 收敛机制
- 操作日志化：所有写操作先形成事件。
- 版本标记：每条事件携带版本向量或逻辑时钟。
- 冲突合并：按字段策略、时间戳策略、域规则策略三层处理。

### 4.3 同步策略
- 在线增量同步：事件流按游标拉取。
- 离线补偿同步：恢复时批量重放。
- 热数据优先：会话与关系优先同步，历史冷数据延后。

## 5. 协议原则（Proto-first）
- 跨端模型、事件、错误码必须在协议层统一定义。
- 协议变更遵循向后兼容优先，破坏性变更需要迁移窗口。
- 任意端实现不得绕过协议定义创建私有主流程模型。

## 6. 数据安全与隐私
- 敏感字段加密存储与传输。
- 最小化同步：仅同步执行所需字段。
- 审计链完整：谁在何时因何目的访问了什么数据。

## 7. 观测与调试
- 每条同步链路可追踪（源节点、目标节点、事件批次）。
- 冲突处理结果可解释并可导出诊断。
- 提供一致性健康度指标（积压、冲突率、重放耗时）。

## 8. 待决策问题
- 冲突收敛是否引入 CRDT 作为默认机制。
- Agent 记忆数据是否与社交数据共享同一同步总线。
- Applet 私有数据是否允许端侧跨设备自动同步。
