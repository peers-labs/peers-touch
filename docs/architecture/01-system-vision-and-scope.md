# 去中心化社交 + AI Agent 系统：愿景与边界

## 1. 文档目的
本文件用于统一新架构的顶层目标、系统边界和阶段性非目标，避免后续设计分散在局部实现层面。

## 2. 系统愿景
- 构建一套可在 Desktop、Mobile、Server 协同运行的去中心化社交 + AI Agent 系统。
- 用户拥有可迁移身份与数据主权，系统具备跨端连续体验与离线容错能力。
- AI Agent 作为一等能力参与内容生产、关系协作与自动化执行。
- Applet 作为扩展机制承载业务创新，但不替代主系统核心能力。

## 3. 设计原则
- 去中心化优先：Server 作为协调节点，不作为唯一真相来源。
- Proto-first：跨端模型、协议和事件统一由协议层定义。
- 端云协同：尽量就近执行，按策略在端、边、云分配任务。
- 安全默认开启：身份、能力、数据访问全部权限化和可审计。
- 渐进演进：先达成最小闭环，再扩展生态能力。

## 4. 系统边界
### 4.1 In Scope
- 多端节点体系：Desktop、Mobile、Server 的角色与协作协议。
- 身份与信任：设备、用户、Agent 的认证、授权、签名链路。
- 社交核心域：账号、关系、会话、内容流与同步机制。
- Agent 核心域：运行时、记忆、工具调用、执行审计。
- Applet 扩展域：运行容器、能力边界、分发与治理。

### 4.2 Out of Scope（首阶段）
- 公有链或代币经济体系设计。
- 全量联邦协议兼容与跨组织互联标准化。
- 全平台一次性全量迁移历史功能。
- 高级商业化模块（广告、复杂结算、付费市场）。

## 5. 目标指标（架构验收）
- 可用性：核心社交流程在单节点或弱网下可降级运行。
- 一致性：同一用户跨端身份、会话、关系状态可收敛。
- Agent 可控性：每次工具调用可追溯、可审计、可中断。
- 可扩展性：新增一个业务扩展不改动核心协议层。
- 研发效率：新能力优先通过标准化扩展机制落地。

## 6. 风险与前置假设
- 假设 Lynx 可作为多端渲染核心之一，但宿主能力适配需自行建设。
- 假设 Server 可持续承担协调职责（索引、中继、策略、审计）。
- 假设团队接受协议先行与分层边界约束。
- 风险点在于能力边界失控、端上实现分叉、协议频繁破坏性变更。

## 7. 待决策问题
- 身份主键采用哪种 DID 方案及轮换策略。
- 数据一致性策略以 CRDT 为主还是 OT + 事件溯源混合。
- Agent 执行默认端侧优先还是服务端优先。
- Applet 的安全模型采用权限白名单还是声明 + 审批双层模型。
