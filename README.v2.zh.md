# Peers-Touch

## 1. 开场共鸣 - 引发思考

### 你的聊天记录，正在训练别人的AI

你在社交平台上的每一次点击、每一条消息、每一个停留，都在被记录、分析、用于训练AI模型。

你没有选择的权利。

这些数据被用来：
- 🎯 精准推送让你上瘾的内容
- 💰 向你推销你可能不需要的商品
- 🤖 训练AI，让它更懂得如何"掌控"你

**这不是科幻，这是现实。**

但，这不应该是唯一的选择。

---

## 2. 我们的答案

### Peers-Touch：属于你的社交网络

**一句话定义**:  
Peers-Touch 是一个让你真正拥有自己数据和隐私的去中心化社交平台。

### 核心价值

🔐 **你的数据，你做主**  
所有数据存储在你自己的设备或服务器上，没有大厂能够访问。

🤝 **连接，不被监控**  
你可以和朋友、家人、社区成员自由交流，没有算法干预，没有广告打扰。

🧠 **AI为你服务，不是监控你**  
你可以选择使用AI助手，但所有AI交互数据都在你的掌控之下。

🌍 **去中心化，但不孤立**  
通过联邦协议，你可以和其他独立站点的用户互动，就像电子邮件一样。

💡 **简单易用，不失先进**  
我们相信，隐私保护不应该以牺牲用户体验为代价。

### 建立一个没有强制性AI和大数据监控的世界

在这个世界中：

- 🤲 **尊重**: 个人信息得到尊重，未经许可不被访问

## 4. 核心理念

### 设计原则

1. **Proto-First** - 类型安全优先
   - 所有数据模型从 .proto 文件生成
   - 确保客户端和服务端数据一致性
   - 避免手动维护模型导致的错误

2. **Privacy-First** - 隐私保护优先
   - 隐私设计贯穿整个开发流程
   - 默认采用端到端加密
   - 最小化数据收集和存储

3. **User-First** - 用户体验优先
   - 简单易用的界面设计
   - 尊重用户的使用习惯
   - 提供清晰的隐私设置

4. **Open-First** - 开放协议优先
   - 基于开放标准和协议
   - 支持跨平台和跨服务互操作
   - 鼓励社区参与和贡献

---

### 技术哲学

1. **简单胜过复杂**
   - 保持代码和架构的简单性
   - 避免过度设计和复杂的解决方案
   - 优先选择成熟稳定的技术

2. **一致性胜过便利性**
   - 统一的代码风格和设计模式
   - 明确的接口和文档
   - 遵循最佳实践

3. **文档化胜过假设**
   - 详细的文档和注释
   - 清晰的设计决策记录
   - 易于理解和维护的代码

4. **模块化设计**
   - 清晰的模块划分和职责
   - 低耦合高内聚的架构
   - 便于扩展和修改

5. **可扩展性**
   - 支持插件和扩展机制
   - 适应未来技术和需求变化
   - 便于社区贡献和定制

## 5. 它是如何工作的

### 核心概念（用人话说）

#### 📮 就像电子邮件一样

还记得电子邮件吗？你可以在 Gmail 注册，你的朋友在 Outlook 注册，但你们依然可以互相发邮件。

Peers-Touch 就是这样：
- 你在 `alice@myfamily.com` 注册
- 你的朋友在 `bob@hisfriends.org` 注册
- 但你们依然可以互相关注、聊天、分享

**这就是"联邦"（Federation）的意思。**

---

#### 🏠 你的数据，你的家

**在传统社交平台上**:
- 你的数据存在 Facebook/Twitter 的服务器上
- 他们可以随意分析、使用、甚至删除

**在 Peers-Touch 上**:
- 你的数据存在**你自己的设备**或**你信任的服务器**上
- 只有你能访问、修改、删除
- 就像你家里的相册，只有你能翻看

---

#### 🔐 私密圈子，真正私密

当你和家人/朋友创建一个私密圈子时：
- 所有消息都经过**端到端加密**
- 即使是服务器管理员也无法查看
- 就像你们在自己家里说悄悄话

---

#### 🤖 AI助手，可选可控

如果你想使用AI助手：
- 你可以选择使用或不使用
- 所有AI交互数据都存储在你的设备上
- 你可以随时查看、导出、删除
- AI为你服务，不是监控你

---

#### 🌐 混合网络：联邦 + P2P

Peers-Touch 使用两种网络模式：

1. **联邦模式**：
   - 用于跨站点通信
   - 基于 ActivityPub 开放标准
   - 就像电子邮件网络

2. **P2P模式**：
   - 用于直接通信
   - 基于 libp2p 协议
   - 更低的延迟，更高的隐私

**为什么混合？**
- 联邦模式提供广泛的连接性
- P2P模式提供更高的隐私和更低的延迟
- 结合两者的优势，提供更好的用户体验

## 6. 你能用它做什么

### 使用场景

#### 👨‍👩‍👧‍👦 家庭内部通信

- **创建家庭私密圈子**，安全分享日常照片和视频
- **不受外部干扰**，没有广告和算法推荐
- **保护孩子隐私**，不用担心孩子的照片被用于AI训练
- **跨设备同步**，手机、平板、电脑都能方便使用
- **端到端加密**，即使服务提供商也无法查看内容

#### 🏘️ 小型社区管理

- **创建独立站点**，为小区、兴趣小组建立专属社交空间
- **自主制定规则**，社区管理由社区成员决定
- **社区数据归属社区**，不会被外部公司控制
- **方便的通知系统**，重要消息及时送达
- **支持多样化内容**，文字、图片、视频、文件等

#### 🎨 兴趣小组交流

- **找到志同道合的朋友**，围绕共同兴趣深入交流
- **专注于内容本身**，不被无关信息打扰
- **保护创作隐私**，草稿和未发布内容完全私密
- **方便的内容归档**，便于查找和回顾
- **支持多种内容形式**，适合不同类型的兴趣爱好

#### 💼 小团队协作

- **安全的团队通信**，敏感信息不会泄露
- **灵活的权限管理**，控制不同成员的访问权限
- **方便的文件共享**，支持多种文件格式
- **无需外部工具**，集成聊天、文件、任务管理等功能
- **数据完全可控**，团队数据掌握在自己手中

#### 🌍 跨社区交流

- **连接不同站点的用户**，就像发送电子邮件一样简单
- **尊重不同社区的规则**，保持社区自主性的同时实现互联互通
- **发现新的兴趣和观点**，扩大社交圈子
- **安全的跨社区互动**，保护用户隐私和安全
